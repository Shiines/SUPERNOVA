# /status — Quick View

## What It Does
Instant snapshot of where things stand mid-session. No full briefing — just the essentials.

## Steps
1. Read `primer.md` (next steps + blockers only)
2. Check active client folder if one is open
3. Output a 5-line status block

## Output
```
STATUS — [time]
Active client: [name or none]
Current phase: [phase number + name]
Next action:   [one thing]
Blocker:       [or "none"]
```
