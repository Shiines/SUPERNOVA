# SUPERNOVA — Rules

## Non-Negotiables
1. One phase at a time — never skip ahead
2. Decisions.md is the source of truth — always log before clearing
3. LITE template before FULL
4. Never invent data — SIRET, addresses, stats = [À compléter]
5. Every deliverable gets saved as a .md file immediately
6. Git push at every /fin — no exceptions
7. Client-facing files never go to /tmp — only working files
8. ultrathink only for high-stakes decisions (pricing, architecture, copy strategy)

## Token Economy
- Never read a full file when you only need a section
- Never re-read a file you just read
- Load 1 workflow phase at a time
- LITE before FULL on all templates
- /clear after each validated phase

## File Naming
- Client files: `05-clients/[client-slug]/`
- Meeting notes: `memory/reunions/CR-YYYY-MM-DD.md`
- Working files: `.tmp/` (disposable)
- Deliverables: named clearly in client folder

## Tone of Voice
- Direct, confident, no filler
- Professional but not corporate
- Short sentences > long paragraphs
- Numbers and proof > adjectives
